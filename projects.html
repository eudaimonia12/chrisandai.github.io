SafeGuardNZ — AI Insurance Claims Platform

University of Auckland × AWS New Zealand · Jul – Nov 2025 · Team lead

AWS Bedrock Amazon Textract Node.js / Express MongoDB Atlas AWS Cognito AWS Amplify
Problem
The client processes thousands of insurance claims a month, each needing manual review of several document types. The team was bogged down re-keying data and validating policies by hand — slowing decisions and hurting customer experience. AWS was already in use for secure document storage, so the brief was to automate classification, extraction and validation while keeping a human in the loop for low-confidence cases.
Stack
Amazon Textract (document OCR), Amazon Bedrock (LLM reasoning) and custom ML models for financial data extraction with confidence scoring; Node.js / Express REST API; MongoDB Atlas; AWS Cognito for auth; deployed on AWS Amplify.
My contribution
Led the five-person team (“Ministry of Iced Tea”) and architected the multi-layer AI system; built the Node.js/Express backend and REST API layer and the auth/data integrations. Delivered guided document upload, real-time extraction of dates & amounts, a confidence-score agent that flags key issues for reviewers, an admin dashboard, automated email notifications and a floating AI support chatbot. Presented to AWS engineers and business stakeholders at the AWS Auckland offices.
Outcome
Won 1st place — selected as the 2025 Semester Two winner of the UoA × AWS Innovation Challenge internship. The platform cuts back-and-forth review time, eliminates manual calculation errors, and scales to handle claim-volume spikes.

Wanderlust — Full-Stack Travel Blog

University of Auckland · COMPSCI 718 / 719 group project · 2024 · Full-stack (Articles lead)

SvelteKit Node.js / Express SQLite TinyMCE Java Swing
Problem
Build a modern blog where travellers can publish, search and discuss adventures — with articles, comments, likes and user accounts — plus a separate desktop admin tool for managing content and users.
Stack
SvelteKit front end (component-based, ~16 pages / 20 components), a Node.js / Express REST API with cookie-based auth middleware, and SQLite behind a DAO layer with Yup validation. A Java Swing app gives administrators user management via the same API.
My contribution
Led the ER / database design, then owned the Articles entity end-to-end: built the Article and ArticleCard components, the articles DAO (SQL JOINs to attach authors) and the full REST API (GET / POST / PATCH / DELETE with auth-guarded routes). Self-taught and integrated the TinyMCE rich-text editor and Multer image uploads.
Outcome
Shipped a working full-stack blog with rich-text authoring, image uploads and article search. A disciplined Git branching strategy (develop / release before main) kept the four-person team's main branch conflict-free throughout.

Spawtify — Community Animal-Spotting Platform

University of Auckland · Group project · 2025 · Full-stack

React Leaflet Tailwind CSS Node.js / Express MongoDB AWS S3
Problem
Neighbourhoods have no shared way to log and find the animals around them — lost pets, wildlife and strays. Existing tools like eBird are built for specialists; Spawtify makes it easy for anyone to record and discover sightings on a map, helping reunite lost pets and build urban-ecology awareness.
Stack
React + React-Leaflet interactive maps styled with Tailwind CSS; a Node.js / Express REST API; MongoDB (Users, Animals, Spots collections); AWS S3 for user photos via pre-signed URLs; JWT + bcrypt auth; a Vitest test suite; auto-deployed to Render on every merge to main.
My contribution
Delivered the platform's core geographic features: dynamic, user-drawn roaming areas on the map (replacing a rigid pre-defined “Areas” model), real-time geocoding to turn addresses into coordinates, and secure cloud image storage — moving uploads from local Multer storage to S3 pre-signed URLs. Wrote backend APIs and Vitest coverage for auth, CRUD and S3/geo handling.
Outcome
A live, map-centric platform where users browse sightings, report animals with photos, and trace pets' roaming areas — deployed and continuously delivered on Render, with a test suite guarding core flows.
